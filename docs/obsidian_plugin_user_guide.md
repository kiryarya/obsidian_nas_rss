# Obsidian プラグイン利用ガイド

## 目的

このガイドは、NAS 上の RSS サーバに接続する Obsidian プラグインの導入方法と日常的な使い方をまとめたものです。

## 前提

- NAS 側の RSS サーバが起動済みであること
- `http://<QNAPのIPアドレス>:43112/health` が応答すること
- Obsidian を利用できること

## 導入の全体像

1. Windows 側でプラグインをビルドする
2. ビルド成果物を Obsidian vault の `.obsidian/plugins/nas-rss-viewer/` に配置する
3. Obsidian でプラグインを有効化する
4. NAS サーバ URL を設定する
5. フィードを追加して使い始める

## 1. プラグインをビルドする

プロジェクトルート:

```powershell
cd C:\Users\tapih\Downloads\Programming\obsidian-rss-nas-platform
pnpm.cmd install
pnpm.cmd --filter @obsidian-rss-nas/obsidian-plugin build
```

ビルド後に使うファイル:

- `packages/obsidian-plugin/main.js`
- `packages/obsidian-plugin/manifest.json`
- `packages/obsidian-plugin/styles.css`

## 2. Obsidian vault に配置する

配置先フォルダを作成します。

```text
<あなたのvault>\.obsidian\plugins\nas-rss-viewer\
```

上で生成または用意した 3 ファイルをこのフォルダにコピーします。

最終的な配置例:

```text
<vault>\.obsidian\plugins\nas-rss-viewer\main.js
<vault>\.obsidian\plugins\nas-rss-viewer\manifest.json
<vault>\.obsidian\plugins\nas-rss-viewer\styles.css
```

## 3. Obsidian で有効化する

1. Obsidian を開く
2. `Settings`
3. `Community plugins`
4. 必要なら `Restricted mode` をオフにする
5. `NAS RSS Viewer` を有効化する

有効化後:

- 左サイドまたはリボンに RSS アイコンが出ます
- コマンドパレットから `NAS RSS ビューを開く` も使えます

## 4. 最初の設定

### 設定画面を開く

1. `Settings`
2. `Community plugins`
3. `NAS RSS Viewer`

### 主要設定

#### サーバ URL

例:

```text
http://192.168.1.50:43112
```

ポイント:

- 末尾の `/health` や `/api/feeds` は付けません
- LAN 内で名前解決できるならホスト名でも構いません

#### 自動再読込間隔

- Obsidian 側で何分ごとに一覧を再取得するか
- `0` にすると自動更新を止められます

#### 未読フィルターを既定で有効化

- 起動直後に未読記事のみ見たい場合はオン

### 接続確認

設定画面の `接続確認` ボタンで、NAS 上のサーバに到達できるか確認できます。

## 5. ビューの使い方

### ビューを開く

方法:

- リボンの RSS アイコンをクリック
- コマンドパレットで `NAS RSS ビューを開く`

### 画面の見方

- 上部ツールバー
  - フィード選択
  - `未読のみ`
  - `あとで読むのみ`
  - `サーバ更新`
  - `フィード追加`
  - `OPML取込`
  - `OPML書出`
  - `表示中を既読`
- 左ペイン
  - 記事一覧
- 右ペイン
  - 選択中の記事詳細

### フィードを追加する

1. `フィード追加`
2. RSS URL を入力
3. 登録後に記事一覧へ反映

注意:

- サーバ側で同じ URL の重複登録は弾かれます

### OPML を一括で取り込む

1. `OPML取込` を押す
2. `.opml` ファイルを選ぶ
3. 追加件数、重複件数、スキップ件数を確認する

補足:

- 既に登録済みの URL は重複としてスキップされます
- 取り込み後、サーバ側で各フィードの初回取得が走ります

### 記事を読む

1. 左ペインで記事をクリック
2. 右ペインに本文またはスニペットが表示される
3. 初回クリック時に未読なら既読へ更新される

### サーバ更新

`サーバ更新` を押すと、NAS 側でバックグラウンド更新が始まります。

ポイント:

- ボタン押下後にすぐ Obsidian の操作へ戻れます
- 更新中はツールバーに状態を表示します
- 更新完了後に一覧が再取得されます

### 既読と未読

- `既読にする`
- `未読に戻す`

どちらも NAS サーバ側に保存されるため、別の端末から見ても状態を共有できます。

### あとで読む

- `あとで読む`
- `あとで読む解除`

これもサーバ側に保存されます。

### 表示中を一括で既読にする

`表示中を既読` を押すと、現在のフィルター条件で一覧に出ている記事をまとめて既読にします。

### ノートとして保存する

記事詳細の `ノート保存` を押すと、Obsidian vault に Markdown ノートとして保存します。

設定項目:

- 保存フォルダ
- 保存テンプレート

テンプレートで使える値:

- `{{title}}`
- `{{date}}`
- `{{link}}`
- `{{feed}}`
- `{{feed_url}}`
- `{{author}}`
- `{{saved_date}}`
- `{{content}}`
- `{{snippet}}`

### 元記事を開く

`元記事を開く` を押すと、記事 URL をブラウザで開きます。

### フィードを削除する

1. 対象フィードを選ぶ
2. 記事を表示した状態で `このフィードを削除`
3. 確認ダイアログで承認

注意:

- そのフィードに紐づく記事もサーバ側から削除されます

## 6. 使い始めのおすすめ運用

### 最初の 10 分でやること

1. サーバ URL を設定する
2. 接続確認を行う
3. 2 から 3 個だけフィードを追加する
4. `サーバ更新` で動作確認する
5. 別端末の Obsidian から同じサーバに接続し、既読状態が共有されるか確認する

### Git コンフリクトを避けるための使い方

- 記事状態はプラグインサーバ側で持たせる
- vault には RSS キャッシュを保存しない
- 記事をノートとして残したい場合だけ、別機能として明示保存する

## 7. よくあるトラブル

### 1. 接続確認に失敗する

確認順:

1. NAS サーバの `/health` が開くか
2. Obsidian の `サーバ URL` に余計なパスが入っていないか
3. ポート `43112` が正しいか
4. NAS のファイアウォールで遮断していないか

### 2. フィード追加が失敗する

考えられる原因:

- URL が RSS ではない
- 対象サイトが一時的に応答していない
- すでに同じ URL を登録済み

### 3. 本文が短い

現在の実装では、RSS の本文またはスニペットを表示します。Web ページ全文抽出は今後の拡張対象です。

## 8. 更新手順

プラグインのソースを更新したら、再度ビルドして `main.js` `manifest.json` `styles.css` を vault 側へ上書きコピーします。
